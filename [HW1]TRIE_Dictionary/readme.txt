POS_tagger

본 과제는 Python 3.6 환경에서 진행하였음.

- POS_tagger 폴더 안 tabular_parsing.py 파일을 실행시키면 test.txt 내의 문장에 대하여 형태소 분석을 수행함.

- 해당 프로그램은 manual_tagging.txt에 저장된 형태소 분석 결과를 자동으로 분석하여 좌우 접속 정보를 grammar.txt에 저장하고 TRIE 기반 사전을 자동으로 생성함. 

- 좌우 접속 정보의 보편적인 활용을 위하여 좌우 접속 정보 작성시에는 형태소 정보만을 고려하였음. 예를 들어 '나/대명사+는/보조사'의 경우, 이를 grammar.txt에 작성할 시에는 '대명사-보조사'와 같은 형식으로 작성하였으며, 프로그램 내부에서도 이와 같은 방식으로 좌우 접속 정보를 활용하였음.

- 정보 탐색의 효율성을 위하여 TRIE 구조 내에 각 형태소에 해당하는 POS tag를 함께 저장하였음. 일반적인 TRIE 구조는 leaf 여부를 1, 0으로만 표시하고 해당 형태소에 대한 POS tag를 따로 사전에서 검색하는 방법을 사용하지만, 이 위치에 POS tag를 직접 저장하였음. leaf일 경우 가능한 모든 tag가 list 형태로 저장되어 있으며, leaf가 아닐 경우에는 None이 저장되는 형태임.
